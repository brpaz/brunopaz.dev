## Features

- **Automatic Dockerfile Linting**: Validates Dockerfiles against best practices and common mistakes
- **CI/CD Integration**: Seamlessly integrates with GitHub Actions workflows
- **Configurable Rules**: Support for custom Hadolint configuration files
- **Clear Error Reporting**: Provides detailed feedback on Dockerfile issues

## Why This Project

Dockerfile quality is crucial for container security and performance. This action helps teams catch issues early in the development process, ensuring Dockerfiles follow best practices before deployment.

## Technical Highlights

- Built as a Docker-based GitHub Action for consistency
- Supports all Hadolint configuration options
- Easy to integrate into existing workflows with minimal setup
- Actively maintained as part of the official Hadolint organization
